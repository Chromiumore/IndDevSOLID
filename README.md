# О программе
## Предметная область
Программа моделирует работу зоопарка:
 * Уход за животными (медосмотр, кормление)
 * Уборку в вольерах
 * Различные должности сотрудников

## Принципы SOLID
### Single Responsibility Principle
Каждый класс имеет определённую узкую зону ответственности, например:
 * Animal - реализация поведения животного
 * Enclosure - моделирует вольер, связывает сотрудников и животных
 * Vet - лечит животных,
 * Keeper - кормление животных, уборка
 * EnclosureCleaner - уборка

### Open/Closed Principle
Классы открыты для расширения, но закрыты для модификаций, например:
 * Animal позволяет добавлять новых животных, не меняя код
 * Employee позволяет добавлять новых сотрудников, не меняя код

### Liskov Substitution Principle
Объекты могут быть заменены наследниками без нарушения работы программы, например:
 * Lion, Parrot, Snake могут использоваться везде, где ожидается Animal
 * Keeper, Vet, EnclosureCleaner могут использоваться как Employee, а также как Feeder, Doctor, Cleaner

### Interface Segregation Principle
Классы не зависят от методов, которые не используют, например:
 * Интерфейсы Doctor, Feeder, Cleaner разделены по обязанностям
 * Keeper реализует Feeder и Cleaner, но не реализует Doctor
 * Vet реализует только Doctor

### Dependency Inversion Principle
Компоненты верхнего уровня не зависят от компонентов нижнего уровня, например:
 *  Enclosure поля doctor, feeder, cleaner объявлены как абстрактные типы (Doctor, Feeder, Cleaner)
 * В вольере не имеет значения, кто именно отвечает за уборку. Это может быть и Keeper, и EnclosureCleaner.


## Ситуации, при которых появятся проблемы добавления нового функционала или расширении бизнес-логики
 * Изменение логики процедур в вольере
 * Добавление нового интерфейса (роли) для сотрудника в Enclosure
 * Создание нового животного с другой логикой кормления и питания требует переопределения методов для каждого нового класса
 * Расширение статистики в Zoo
 * Потребность в проведении процедур в разное время

## Улучшения в качестве дизайна
 * Лёгкое добавление новых классов для животных и сотрудников
 * Зависимость от абстракций позволяет использовать различные реализации
 * Единые интерфейсы устраняют дублирование кода
